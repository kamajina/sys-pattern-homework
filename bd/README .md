# Домашнее задание к занятию "`bd`" - `grr`


### Задание 1

```
1.1. Бюджетирование, отчёты, прогнозирование рисков
Лучше подойдёт реляционная (SQL) СУБД (например, PostgreSQL, Oracle). Она обеспечивает ACID-транзакции, чёткую структуру, целостность данных и сложные аналитические запросы, что критично для финансовой отчётности.

1.2. Лендинги и CRM

Для лендингов: Документная NoSQL СУБД (например, MongoDB). Позволяет быстро и гибко хранить данные по лидам без жёсткой схемы.

Для CRM: Реляционная СУБД (например, PostgreSQL) или гибридная CRM-система. Нужна структурированность для сделок, клиентов и аналитики продаж.

1.3. База знаний отдела контроля качества
Реляционная СУБД (например, SQLite или PostgreSQL) с иерархической структурой таблиц. Позволит создать простую и понятную систему категорий (нормы, правила, материалы).

1.4. Логистика и маршруты доставки
Графовая СУБД (например, Neo4j). Оптимальна для работы со связями (точки, маршруты, курьеры), быстрого расчёта кратчайших путей и распределения ресурсов.
```

### Задание 3

```
3.1. Преимущества SQL перед NoSQL

ACID-гарантии — надёжность транзакций.

Чёткая структура данных через схему.

Мощный язык запросов (JOIN, агрегации).

Целостность данных благодаря внешним ключам.

Зрелость и стандартизация — проверенные решения.
```

### Задание 4

```
Критерий выбора СУБД: способность работать в распределённом режиме на 1000 машинах с поддержкой параллельных вычислений и отказоустойчивости.
Лучшая модель: MapReduce (например, в Hadoop/Spark) или MPP-архитектура (как в ClickHouse или Greenplum).
Почему: MapReduce позволяет распределять вычисления на множество узлов, обрабатывая большие данные параллельно, что эффективно для сложных аналитических задач.
```
