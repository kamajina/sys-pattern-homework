### Задание 1
1.1. Восстановление данных в полном объёме за предыдущий день
Подходит полное резервное копирование, выполняемое ежедневно. Этот метод создаёт полную копию всех данных на момент создания бэкапа. 

1.2. Восстановление данных за час до предполагаемой поломки

Полный бэкапа (например, раз в сутки или неделю).
Журнал транзакций (WAL в PostgreSQL, binlog в MySQL), который постоянно архивируется.
При аварии можно восстановить полный бэкап, а затем применить журналы транзакций до момента за час до поломки. Это обеспечивает точное восстановление на любой момент времени.

### Задание 2
Резервирование:

`pg_dump -U username -F c -b -v -f backup.dump mydatabase`


Восстановление:

`pg_restore -U username -d newdatabase -v backup.dump`

### Задание 3

Создать полный бэкап:

`mysqldump -u root -p --flush-logs --master-data=2 --single-transaction --all-databases > full_backup.sql`

Копировать бинарные логи для инкрементного бэкапа:

`mysqlbinlog /var/lib/mysql/mysql-bin.000012 > inc_backup_000012.sql`
